"SPARQLPRESS"

"SPARQLPRESS" - have a RDF store to a blog or a community site, so that all content which is created, is also put in the store - so each site has a companion RDF store that is being worked on in Drupal - people can query directly the content of drupal, but not in a SQL way but in a SPARQL way that is a research or implemenatation question - when will you run into problems when people run expensive querys. DOS you could have limits on queries or have preset querys that people can run - like a web service

Miscellaneous

SIOC Discovery

Four possible architectures (also see Whiteboard 1):

  • True Peer-to-peer
  • P2P with SuperNodes
  • Centrally stored - like Sindice
  • Store data in the applications end-user-oriented (e.g. an application like a feed reader collects only the data the user wants)

Add SIOC apps to SemWeb catalogs

Adding SIOC applications to semantic web catalouges - which are there?

  • If anyone sees a wiki where SIOC would belong - add it.
  • Everyone should create a doap.rdf files for their SIOC projects. It needs a foaf:topic_interest pointing to SIOC on DBpedia (although foaf:topic_interest has foaf:Person for the domain). Send the link of your doap file to Ping the semantic web and Alex' DOAP store will get it.
  • Alex agrees to nag people about creating DOAP files.
  • There is this semantic web education and outreach on ESW wiki and there is only openlink as SIOC application → change that.
